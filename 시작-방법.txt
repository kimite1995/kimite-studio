KIMITE STUDIO 시작 방법 (초보자용 - 2026.06)

==================================================
★ 가장 쉬운 방법 (추천)

1. 메인 사이트 키기 (포트폴리오 보는 곳, localhost:3000)
   → start-main.bat 파일을 더블클릭하세요.

2. 관리자 사이트 키기 (작품 올리고 수정하는 곳, localhost:3333)
   → start-admin.bat 파일을 더블클릭하세요.

※ 두 개를 동시에 켜야 합니다. (두 개의 검은 창이 떠 있어야 해요)

==================================================
문제 해결 체크리스트 (초등학생도 이해할 수 있게)

【문제 1】 로컬에서는 작품이 안 보여요

1. start-admin.bat 을 더블클릭해서 관리자 창을 켜세요.
2. 브라우저에서 http://localhost:3333 들어가기
3. 로그인하기
4. 왼쪽에서 "Content" 또는 "작품" 클릭
5. 내가 올린 작품이 목록에 있나요?
   - 있으면 → 그 작품을 클릭
   - 오른쪽 위에 큰 "Publish" 버튼이 보이면 → 그 버튼을 꼭 누르세요!
   - Publish를 누르지 않으면 메인 사이트에 절대 안 나와요.

6. Publish를 눌렀으면, start-main.bat 으로 메인 사이트도 다시 켜거나 새로고침 (Ctrl + Shift + R)

【Vision으로 데이터 확인하는 법】
1. 관리자(http://localhost:3333)에 들어감
2. 위쪽 메뉴에서 "Vision" 클릭
3. 왼쪽 빈칸에 아래 글을 정확히 복사해서 붙여넣기:

*[_type == "work"] | order(_createdAt desc) {
  _id,
  title,
  shortDesc,
  category
}

4. 오른쪽 위 "Run" 버튼 클릭
5. 결과가 나오나요?
   - 작품이 1개 이상 나오면 → 데이터는 잘 들어간 거예요.
   - 아무것도 안 나오면 → 아직 Publish를 안 했거나 작품이 저장이 안 된 거예요.

【문제 2】 배포된 사이트 (https://kimite-studio.vercel.app/) 에 수정한 게 안 보여요

이유: 우리가 로컬에서 고친 파일(모바일 글씨 크기, 줄바꿈 등)을 인터넷(베르셀)에 아직 올리지 않았기 때문입니다.

해결 방법 (차근차근):

1. Git Bash 또는 PowerShell을 열고 프로젝트 폴더로 이동
   cd "C:\Users\박상우\kimite-studio"

2. 아래 명령어를 한 줄씩 복사해서 붙여넣고 엔터 치세요.

   git add .

   git commit -m "모바일 개선 + 관리자 bat 파일 추가 + Sanity 데이터 연동 수정"

   git push

3. git push가 성공하면, Vercel 사이트에 자동으로 새 버전이 배포됩니다.
   (보통 1~3분 정도 걸려요)

4. https://kimite-studio.vercel.app/ 에 가서 강력 새로고침 (Ctrl + Shift + R) 해보세요.

【Vercel에 환경변수 넣는 법 (매우 중요! 데이터가 안 나올 때)]

배포된 사이트에서 데이터가 안 나오는 가장 흔한 이유는 Vercel에 환경변수가 없어서예요.

1. 브라우저에서 https://vercel.com 접속해서 로그인
2. 본인 프로젝트 "kimite-studio" 클릭
3. 상단 메뉴에서 "Settings" 클릭
4. 왼쪽 메뉴에서 "Environment Variables" 클릭
5. 아래 두 개를 추가해야 해요:

   Name: NEXT_PUBLIC_SANITY_PROJECT_ID
   Value: srn2jla0
   Environment: Production, Preview, Development 모두 체크

   Name: NEXT_PUBLIC_SANITY_DATASET
   Value: production
   Environment: Production, Preview, Development 모두 체크

6. Add 버튼 클릭
7. 다시 "Deployments" 탭으로 가서 가장 최근 배포를 찾아서 "Redeploy" 버튼 클릭

이렇게 하면 배포된 사이트도 Sanity 데이터를 볼 수 있게 됩니다.
